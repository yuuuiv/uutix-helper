# UUTIX Helper

UUTIX Helper 是一个用于 UUTIX 购票流程的 Tampermonkey 用户脚本。它会在页面右侧显示一个悬浮控制面板，按用户设置的场次、票价位置、数量、回流策略和支付方式执行自动选择，并在后续流程中尽量连续完成提交订单、确认条款、选择付款方法和进入支付。

## 功能

- 在活动详情页等待入口按钮变为可购买状态后自动进入购票流程。
- 在购票流程页按位置选择场次、票价和数量。
- 使用 DOM 变化唤醒和短轮询兜底等待票价列表、按钮和页面跳转，减少固定等待时间。
- 对关键按钮使用短时间补点，提高抢票场景下的触发容错。
- 可开启「抢回流」模式：当目标票价显示「暂时无票」或「暫時無票」时，按可调间隔持续点击「购买」或「購買」，等待回流票进入下一步。
- 遇到购票人数过多页面时，会按冷却策略尝试刷新重试。
- 进入购物车后自动点击「提交訂單」或「提交订单」。
- 进入交易预览页后自动勾选购票条款，并点击「立即支付」。
- 进入支付页后按预设付款方法选择对应支付方式，确认目标方式已选中后再点击「確認支付」或「确认支付」。
- 选择银行卡支付时，可临时填写持卡人、卡号、有效期和 CVV，并在支付页尝试自动填入。
- 提供可拖拽、可隐藏的悬浮窗，位置和隐藏状态会保存到本地。
- 使用 `localStorage` 保存自动续跑状态，支持页面跳转后继续执行后续步骤。

## 文件

主要脚本文件：

```text
UUTIX Helper (v18).js
```

内容是用户脚本格式，可以作为 Tampermonkey 脚本导入或复制使用。

## 安装

1. 安装 Tampermonkey 或兼容的用户脚本管理器。
2. 新建脚本。
3. 将 `UUTIX Helper (v18).js` 的全部内容复制到脚本编辑器。
4. 保存并启用脚本。
5. 打开 UUTIX 相关页面，确认右侧出现 `UUTIX v18` 悬浮窗。

## 使用

1. 在悬浮窗中设置：
   - 场次位置：从上到下第几个场次。
   - 票价位置：从上到下第几个票价。
   - 目标数量：需要购买的门票数量。
   - 是否抢回流：默认关闭；开启后会把「暂时无票」视为可继续尝试的回流票状态。
   - 回流间隔：抢回流模式下点击购买按钮的间隔，默认 `180ms`。
   - 支付方式：后续支付页要选择的付款方法。
   - 银行卡信息：仅选择卡支付时需要填写。
2. 点击「开始」。
3. 脚本会根据当前页面自动选择合适流程：
   - 详情页：等待购买入口并进入购票流程。
   - 购票流程页：选择场次、票价、数量并点击购买。
   - 购物车页：点击提交订单。
   - 交易预览页：勾选条款并点击立即支付。
   - 支付页：选择预设付款方法，确认选中后再确认支付；如出现银行卡表单，会尝试填入卡信息。
4. 如需中止，点击「停止」。

## 卡信息隐私

- 脚本不会把卡号、CVV、有效期或持卡人写入 `localStorage`。
- 脚本不会把卡号、CVV、有效期或持卡人写入用户脚本管理器的长期存储。
- 这些信息只会在当前标签页临时传递给支付页，支付页读取后会立即清除临时传递内容。
- 浏览器自身的自动填充或密码管理器不受脚本控制，如不希望浏览器保存，请在浏览器设置中关闭相关功能。

## 悬浮窗

- 拖拽标题栏可以移动悬浮窗。
- 点击「隐藏」可收起悬浮窗。
- 收起后页面会保留一个 `UUTIX` 小按钮，点击即可展开。
- 悬浮窗位置和隐藏状态保存在浏览器本地，下次打开页面会自动恢复。

## 自动续跑

脚本使用浏览器本地存储记录必要状态：

- 自动进入购票流程。
- 页面跳转后继续提交订单。
- 页面跳转后继续处理交易预览页。
- 页面跳转后继续处理支付页。
- 保存上次填写的场次、票价、数量、回流间隔和支付方式。
- 不保存银行卡敏感信息。

点击「停止」会清理自动续跑状态。

## 调试

脚本会在浏览器控制台输出带有 `[UUTIX Helper]` 前缀的日志。排查问题时建议同时观察：

- 悬浮窗状态文字。
- 浏览器控制台日志。
- 页面是否仍在加载。
- 目标场次、票价和数量是否与预期一致。
- 支付方式是否与预期一致。
- 抢回流间隔是否过快导致频繁触发平台拥挤或限制页面。
- 按钮文字是否发生变化。
- 是否出现验证码、登录过期、库存不足、支付限制或外部支付页限制。

## 维护说明

页面结构变化时，优先检查这些区域：

- 购买入口按钮识别。
- 场次列表识别。
- 票价列表识别。
- 数量选择器识别。
- 购物车提交按钮识别。
- 交易预览页条款勾选框识别。
- 立即支付按钮识别。
- 支付页付款方法识别。
- 支付页付款方法选中态识别。
- 支付页确认支付按钮识别。
- 银行卡输入框识别。

维护时建议保持现有主流程不变，只针对失效的 DOM 选择器和状态判断做小范围适配。

## 注意事项

- 脚本只负责辅助页面操作，不保证库存、订单或支付结果。
- 使用前请确认目标场次、票价位置、数量和支付方式设置正确。
- 抢回流间隔过低可能更容易触发平台拥挤或限制页面，建议从默认值开始调整。
- 如果页面出现验证码、登录失效、网络错误或风控限制，需要手动处理。
- 银行卡信息自动填入后，请在继续支付前自行核对。
- 请遵守 UUTIX 平台规则以及当地适用法律法规。
